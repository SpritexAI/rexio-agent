# Project Rules for RexiO Agent

- **Human-written Aesthetics:** All code, inline comments, variable names, and documentation must read as if they were hand-crafted by an experienced human software engineer. 
- **No AI Signatures:** Never include placeholders, comments, or headers that suggest code was generated by an AI assistant (e.g., "AI-generated", "Auto-generated by assistant", robotic logging statements).
- **Production-ready Code:** Follow PEP 8 guidelines, use clean type-hinting, standard docstring formats, and write meaningful developer comments only where complex logic requires explanation.
